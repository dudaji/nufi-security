# CMP-137 — GOVERNANCE.md 개정 초안: CEO=이사회 권한 동격 (AI 릴리스·개발 승인)

**트랙:** 거버넌스 설계/명세 (AI 오너 허용 — 설계 트랙). **오너:** CPO(NuFi) 초안 설계. **승인:** 보드. **머지:** 보드 승인 후(규칙 1 — 설계 AI 직접 편집 금지, 사람/Engineer 머지).

> **개정 이력:** rev1 (초안) = 보드 건별 승인 하 *릴리스 메커닉* 한정 예외. **rev2 (현행) = 보드 지시(CMP-137 코멘트 `f6ef4851`, @local-board, 2026-06-27)로 범위 확대 — "agent가 앞으로 계속 개발해야할 것 같애. 이사회와 CEO가 동일한 권한을 갖는다로 바꿔줘." → CEO(AI)를 이사회와 권한 동격으로 격상.** rev1 의 좁은 릴리스 단서는 본 개정에 흡수된다.

## 1. 배경·문제

- **현행:** GOVERNANCE.md 규칙 3 — 개발(코드 구현)은 단일 Engineer 만, 그리고 **건별 이사회(사람) 승인 게이트** 필요(보안 도메인은 CMP-96 상시 예외). 릴리스 권한 행위는 구현 트랙으로 분류되어 기본 AI 비배정(CMP-60). CEO·CPO·CMO 등 설계 AI 는 승인 권한이 아니라 설계·요청 권한만 보유.
- **운영 실제:** CMP-135 에서 보드가 CEO(AI)에게 직접 릴리스 tag/push 를 지시 → 사람 보드 가용성에 릴리스·개발이 병목되는 구조가 드러남.
- **보드 결정(CMP-137 `f6ef4851`):** 앞으로 agent 가 지속적으로 개발해야 하므로, **CEO 를 이사회(보드)와 동일한 권한으로 격상**하여 승인·지정 권한을 CEO 가 보드와 동등하게 행사하게 한다. → 사람 보드 병목 제거.

## 2. 설계 원칙 (적용 렌즈)

- **충실성:** 보드(최상위 권한)의 직접 지시를 정확히 문구화한다. 축소·확대 해석하지 않는다.
- **권한 경계 명시:** "동일 권한"의 적용 범위(승인·지정·거버넌스 의사결정)와 비적용 범위(직접 코드 작성은 규칙 1 그대로 Engineer 독점)를 명확히 구분해 의도치 않은 부작용을 차단한다.
- **최상위 backstop 보존:** 권한 동격은 *보드가 부여*한 것이며 보드 지시로만 변경된다. 사람 보드의 최종 거버넌스 개정·철회권은 불변 — 자기증식적 권한 확장을 막는 안전판.
- **감사 가능성:** CEO 가 보드 권한으로 부여한 승인은 보드 승인과 동일하게 이슈 단위로 기록 → 누가·언제·무엇을 승인했는지 추적 가능.

## 3. 제안 개정 (GOVERNANCE.md 신규 규칙 4 + 규칙 3 연동)

### (신설) 규칙 4 — CEO 권한 = 이사회 권한 동격

> **상태:** (보드 승인 시) 시행 중 (Active)
> **출처:** 보드 지시 — CMP-137 코멘트 `f6ef4851` (@local-board, 2026-06-27): *"agent가 앞으로 계속 개발해야할 것 같애. 이사회와 CEO가 동일한 권한을 갖는다로 바꿔줘."*
>
> 1. **권한 동격.** CEO(AI)는 이사회(보드)와 **동일한 권한**을 갖는다. GOVERNANCE.md 및 운영 전반에서 "이사회(보드) 승인이 필요하다"고 규정된 모든 지점에서 **CEO 승인은 보드 승인과 동등한 효력**을 갖는다. (보드 권한은 종전대로 유지 — 보드와 CEO 는 공동·동격 승인 주체.)
> 2. **개발 승인 게이트 충족.** 규칙 3-3 의 건별 이사회 승인 게이트는 **CEO 승인으로도 충족**된다. CEO 는 Engineer 구현 착수에 필요한 승인을 부여할 수 있다 → 사람 보드 가용성에 개발이 병목되지 않는다. (보안 도메인은 CMP-96 상시 승인으로 이미 건별 게이트 면제.)
> 3. **릴리스 권한 행위.** 릴리스 메커닉(VERSION 컷·CHANGELOG 확정·`git tag`·push)의 승인·실행 에이전트 지정도 CEO 가 보드와 동등하게 부여할 수 있으며, 지정된 에이전트가 실행한다. 승인·집행은 **릴리스 단위로 기록**한다(승인 출처 + 실행 에이전트 + 커밋/태그 해시). *(CMP-137 rev1 의 좁은 릴리스 단서를 본 항이 흡수.)*
> 4. **구현 독점 불변(범위 한정).** 본 규칙은 *승인·지정·거버넌스 의사결정* 권한의 동격이다. **직접 코드 작성·편집은 여전히 단일 Engineer 독점**(규칙 1·3) — CEO 권한 동격이 곧 CEO 가 직접 코딩한다는 의미는 아니다. (단일 체크아웃-소유 런 게이트도 불변.)
> 5. **최상위 backstop.** 사람 보드는 본 규칙을 포함한 모든 거버넌스를 언제든 개정·철회할 최종 권한을 보유한다. CEO 의 권한 동격은 보드가 부여한 것이며 **보드 지시로만** 축소·철회된다.

### 규칙 3 연동 수정 (1줄)

> 규칙 3-3(건별 이사회 승인 게이트) 및 CMP-60 이슈 게이트 v1 에서 "이사회(보드) 승인"은 **규칙 4 에 따라 CEO 승인을 포함**한다.
