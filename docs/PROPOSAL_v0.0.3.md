**끝남(Adopt — 채택 마찰 해소, CMP-119~125 done):**

- 무재기동 룰 **핫리로드(단건)+드라이런** + retain_raw/키회전 하드닝 — CMP-124

**남김(Operate 공백 — 본 제안의 표적):**
- **감사 가시성 화면 0** — 결정·해시체인·우회를 사람이 읽을 read-only 화면 없음(저장소에 `dashboards/` 없음, Langfuse/Grafana 미배선). v0.0.2 에선 "Could" 로 비포함.
- **상시 커버리지 보증 0** — `nufi doctor` 는 *1회* 진단. "지금 이 순간 내 트래픽 중 몇 %가 실제로 게이트웨이를 통과했나"의 **런타임 상시 측정/알림**은 없음(`capture/flow_tap.py` 는 탐지 훅만 보유).
- **정책 운영 규모화 미흡** — 핫리로드는 단건/단일프로파일(CMP-124). 멀티프로파일·라우트별 바인딩·버전/롤백 미지원.
- **🚩 v0.0.2 가 릴리스되지 않음(릴리스 위생 부채)** — 자식 이슈는 전부 done 이나 **`VERSION`=0.0.1, CHANGELOG 최상단=0.0.1, git tag 는 `v0.0.1` 뿐.** v0.0.2 태그가 컷되지 않아 사용자가 버전을 핀할 수 없음. (참고: `docs/PROPOSAL_v0.0.2.md`·`STATUS.md` 일부 편집이 미커밋 상태일 수 있음 — 릴리스 컷 시 함께 정리.)

→ **결론:** Adopt 가 "반나절에 채택·검증"을 풀었다면, Operate 의 바인딩 제약은 **"운영 중 무엇이 실제로 지켜지는지 보이지 않는다"** — 즉 *관측(Observe)·보증(Assure)·운영(Operate)*. 세 단어가 v0.0.3 의 세 기둥입니다. 그 앞에 **릴리스 위생(선행)** 을 둡니다.

## 1. v0.0.3 후보 방향 5종 (각각 하나의 응집된 트랙)

각 트랙 = 무엇 / 왜 / 대표 산출물 / Effort. 보드가 조합을 고를 수 있게 5종 제시. 구현 오너=Engineer(CMP-96), **단 O5 는 사람 오너**(CMP-60 구현 트랙).

### O1 — Audit Observability (감사 가시성·관측)  ← Operate 의 *관측*
- **무엇:** read-only 감사 대시보드(Langfuse 또는 Grafana 위) — ① 결정 뷰어(차단/가명화/경고 이벤트 조회·필터) ② 감사 해시체인 무결성 뷰 ③ 우회 탐지 이벤트 타임라인 ④ 카테고리별 탐지량 추이. **쓰기 권한 없음**(전략·관측 전용, 프로덕션 무변경).
- **왜:** "보안팀이 온프렘에서 *관측*". 지금은 감사가 100% 적재되지만 *사람이 볼 화면이 0* — 운영 신뢰의 직접 장애물. 대부분 LLM 가드레일은 결정 가시화를 안 줌 → 차별점.
- **대표 산출물:** `dashboards/` (Langfuse 프로젝트 or Grafana 대시보드 JSON + 데이터소스 어댑터), 결정/해시체인/우회 패널 3종, 가이드 1절.
- **Effort:** M

### O2 — Coverage Assurance (커버리지 보증·상시)  ← Operate 의 *보증* / `nufi doctor` 의 상시 런타임판
- **무엇:** ① **커버리지 리포트** — "내 트래픽 중 X%가 실제 게이트웨이를 통과" 를 런타임에 상시 집계(`capture/flow_tap.py` 연계) ② **우회 상시 모니터링/알림** — 게이트웨이를 우회한 outbound 를 준실시간 탐지해 알림 발화 ③ doctor 의 카나리 검증을 주기 스케줄로.
- **왜:** 하이브리드 #1 실패 모드=*조용한 우회*. nftables 집행을 **측정 가능한 보증**으로. v0.0.2 의 1회 doctor 를 상시 감시로 승격 — Operate 의 핵심 차별점.
- **대표 산출물:** 커버리지 집계기 + 리포트 명령(`nufi-egress coverage` 류), 우회 알림 룰/싱크, 임계 알림.
- **Effort:** M

### O3 — Policy Ops at Scale (정책 운영 규모화)  ← Operate 의 *운영*
- **무엇:** CMP-124 단건 핫리로드를 규모화 — ① **멀티프로파일** 동시 운영 + **라우트/테넌트별 프로파일 바인딩** ② 정책 **버전·드리프트·롤백**(무재기동) ③ 변경 감사(누가·언제·무엇을).
- **왜:** 단일 프로파일·단건 리로드로는 다중 워크로드 운영 불가. 운영자가 안전하게 정책을 굴리는 1급 경험.
- **대표 산출물:** 프로파일 레지스트리 + 바인딩 규칙(`routing.yaml` 확장), 버전/롤백 명령, 변경 감사 로그.
- **Effort:** M

### O4 — Detection Depth / DLP (탐지 깊이 · 수요 조건부)
- **무엇:** 기밀 3계층 **IDM 문서지문**(인덱싱된 문서 일치 — EDM 의 비구조화 확장). 임베딩 분류기(패러프레이즈 일반화, 라벨 필요)는 **그 다음** 단계.
- **왜:** 차별화된 DLP. **단 디자인파트너 수요 신호 의존** — 미확인 선투자 비권장. v0.0.2 에서도 Won't 였음.
- **대표 산출물:** (수요 확인 시 별도 설계) IDM 인덱서 + 매칭기.
- **Effort:** L–XL

### O5 — Release Hygiene (릴리스 위생 · 선행)  ★먼저
- **무엇:** **v0.0.2 릴리스 태그 컷** — `VERSION`→0.0.2, CHANGELOG 0.0.2 섹션(CMP-119~125·127 변경 정리), `git tag v0.0.2` + 푸시. 미커밋 문서(`PROPOSAL_v0.0.2.md`·`STATUS.md` 편집) 동반 정리. 릴리스 오너 지정.
- **왜:** v0.0.2 작업은 끝났으나 **릴리스 자체가 안 됨** — 사용자가 버전 핀 불가, 다운스트림 혼선. v0.0.3 을 쌓기 전에 베이스라인을 고정해야 함.
- **오너:** **사람(릴리스 오너)** — 태그 컷·VERSION/CHANGELOG 편집·푸시는 구현 트랙이므로 CMP-60 에 따라 AI 에이전트 비배정. (CPO 는 CHANGELOG 초안 *설계*만 제공 가능.)
- **Effort:** S

## 2. 로드맵 갱신 (Horizon 재배치)

```
            │ Adopt (v0.0.2 ✅)      │ Operate (v0.0.3 ← 본 제안)  │ Scale·Differentiate (v0.1.0)
────────────┼───────────────────────┼─────────────────────────────┼──────────────────────────────
 관측       │ (감사 적재만)          │ O1 감사 대시보드(read-only)  │ 멀티테넌트 콘솔
 보증       │ doctor=1회 진단        │ O2 커버리지 보증(상시·알림)  │ D5 SLA 리포팅
 운영       │ 핫리로드 단건(CMP-124) │ O3 정책 규모화(멀티프로파일) │ 멀티테넌시·RBAC
 탐지깊이   │ —(수요 관망)           │ O4 IDM 문서지문(수요 시)     │ 임베딩 분류기(수요 시)
 위생       │ —                      │ O5 v0.0.2 태그 컷(선행)      │ 릴리스 자동화
────────────┴───────────────────────┴─────────────────────────────┴──────────────────────────────
 릴리스 주제 │ "반나절에 채택·검증"   │ "보안팀이 온프렘에서         │ "다고객·차별화 DLP·규모 SLA"
            │                       │  운영·관측·보증"             │
```

호라이즌은 순서지 마감일이 아닙니다. O4(탐지깊이)는 **디자인파트너 수요 신호가 뜨면** 어느 호라이즌으로든 당겨집니다.

## 3. v0.0.3 권장 구성 (CPO 추천 · MoSCoW)

Operate 테마(관측·보증·운영)에 정확히 매핑되도록 O1·O2 를 척추로, O3 를 병행, O5 를 선행 게이트로 둡니다.

| 등급 | 포함 | 근거 (도메인 렌즈) |
|---|---|---|
| **Must(선행)** | **O5** v0.0.2 태그 컷 | *릴리스 위생*: 베이스라인 미고정. 오너=사람. v0.0.3 의 전제 |
| **Must** | **O2** 커버리지 보증(상시·알림) | *보증*. 하이브리드 #1 실패=조용한 우회 → Operate 핵심 차별점 |
| **Must** | **O1** 감사 대시보드(read-only) | *관측*. 운영 신뢰의 직접 장애물(사람이 볼 화면 0) |
| **Should** | **O3** 정책 운영 규모화 | *운영*. 다중 워크로드 운영의 1급 경험 |
| **Could / 조건부** | **O4** IDM 문서지문 | 디자인파트너 수요 확인 시 당김 — 미확인 선투자 비권장 |
| **Won't(이번)** | 멀티테넌시·RBAC·TLS MITM·임베딩 분류기 | 범위·리스크·수요 — v0.1.0 Scale 호라이즌 |

**시퀀스:** **M0 Release Cut**(O5, 사람 오너) → **M1 Observe+Assure**(O1+O2) → **M2 Operate-at-Scale**(O3) → *(조건부 M3 DLP, O4 — 수요 시)*.

각 마일스톤 binary 완료기준:
- **M0:** `VERSION`=0.0.2 · CHANGELOG 0.0.2 섹션 존재 · `git tag v0.0.2` 푸시 완료(릴리스 오너).
- **M1:** 대시보드에서 감사 결정 1건 + 해시체인 무결성 1건을 read-only 조회 · 커버리지 리포트 1건("X% 게이트웨이 통과") 산출 + 우회 시나리오 1건에 알림 발화.
- **M2:** 멀티프로파일 2개 동시 운영 + 라우트별 바인딩 1건 · 정책 버전 롤백 1건을 무재기동으로 수행.
