# NuFi 로드맵 제안 — 규제 준수 증빙 게이트웨이

> 상태: **채택됨 (2026-06-29)** — 보드가 집중 2축(①규제 준수 증빙 게이트웨이 + ②한국어 PII 이그레스 방화벽)을 선택. 근거 분석은 [금융분야 인공지능 보안 안내서(2026.6) 분석](research/FSEC_AI_GUIDE_2026.md), 옵션 비교는 [솔루션 집중분야](research/SOLUTION_FOCUS_OPTIONS.md).
> 본 문서의 우선순위·시퀀싱은 채택된 2축을 전제로 한다. 잔여 결정(차기 베팅, 목표 연결, 구현 담당자)은 리더십 정합 대기.

---

## 1. 포지셔닝

> **NuFi = 금융권 생성형 AI 도입 시 「금융분야 인공지능 가이드라인」 제7장(7.2 공격 탐지·7.3 자산 보호·7.5 기존 보안 확장·7.6 검증 운영)과 망분리 혁신금융서비스 보안대책을 충족시키는 온프렘 게이트웨이.**

구매 트리거가 **"데이터 유출 걱정" → "혁신금융 지정 부가조건(규제) 준수 증빙"**으로 격상된다. 상용 생성형 AI의 내부망 이용은 망분리 예외 적용을 위한 혁신금융서비스 지정이 전제이고, 그 지정에는 강화된 보안대책 평가가 부가조건으로 부과된다(점검항목 36). NuFi의 기존 차별점 — 한국어 PII 처리·온프렘·에어갭 — 이 그대로 이 규제 요건과 맞물린다.

## 2. 전략 근거

안내서의 입출력 필터링·감사·자산 무결성 축을 NuFi가 거의 1:1로 구현한다. 분석 문서 §5 커버리지 매트릭스 기준:

- **이미 직접 충족:** 입출력 PII 탐지·차단(⑦), 출력 노출 방지(⑧·망 2.5), 국외 이전 차단(26), 자동화 접근 모니터링·최소권한(24), 입출력 로그(망 1.2), 위변조 방지 형상관리(망 2.7).
- **부분 충족 → 보강:** 회피 공격 탐지(⑥·⑨), 요청 횟수 제한(⑫·망 2.6), 실시간 모니터링·점검·보고(⑪·32~35), 자산 무결성·킬스위치(⑭·⑰·25).

따라서 로드맵의 첫 슬라이스는 **새 기능 개발이 아니라 이미 충족한 통제를 규제 언어로 증빙**하는 것이어야 ROI가 가장 높다.

## 3. 우선순위 (Value / Effort)

| 우선순위 | 항목 | 규모 | 핵심 가치 |
|---|---|---|---|
| **P0** | 컴플라이언스 매핑 리포트 | S | 점검표 자동 충족 증빙 → 도입 결정·규제 대응 직결 |
| **P1** | 사용자·테넌트별 요청 횟수 제한(rate limit) | M | 안내서가 "반드시 적용" 명시(망 2.6) |
| **P2** | 생성형 회피 공격 인라인 탐지 | L | 한국어 가드모델 차별화 |
| **P3** | 킬스위치·자산 무결성 인벤토리 명시화 | S~M | 운영 안전장치 완성 |

### P0 — 컴플라이언스 매핑 리포트 (S, 최우선)

기존 `nufi-egress report compliance` 를 확장해 38개 점검항목·망분리 평가기준에 대한 NuFi 통제 커버리지를 **자동 산출**한다. 직접 충족 항목은 기존 리포트 증빙(무결성·차단/가명화 건수·결정 로그)에서 상태를 자동 판정하고, 부분/범위 밖 항목은 정적 라벨로 분류한다.

- **목표 산출물:** 점검항목 → NuFi 통제 매핑표 + 충족 상태(충족/부분/범위 밖) + 증빙 출처. *"NuFi 도입 = 점검표 자동 충족 증빙"*.
- **충족 점검항목:** ⑦·⑧·24·26 및 망 1.2·2.4·2.5·2.7·3.1 (직접) + 보강 항목(부분, 후속 트랙 링크).
- **종료코드:** 기존 무결성 게이트(0 정상 / 1 변조) 유지. 커버리지는 정보성으로, 신규 비-0을 도입하지 않는다.

### P1 — 요청 횟수 제한 (M)

안내서는 ⑫·망 2.6에서 *"이용자의 생성형 AI 요청 및 결과 출력 횟수를 일정 수준 이하로 제한"*하고, 생성형 AI에는 데이터·모델 추출 방어와 자동화 공격 리소스 과다 사용 방지를 위해 **요청 횟수 제한을 반드시 적용**할 것을 명시한다. 테넌트/사용자별 rate limit을 게이트웨이에 추가한다(배치 업무의 정상 대량 요청은 상향 조정 가능하게).

### P2 — 생성형 회피 공격 인라인 탐지 (L)

⑥·⑨(생성형)의 프롬프트 인젝션·탈옥 탐지를 입출력 필터에 인라인으로 확장한다. 안내서 권고대로 별도 가드모델을 연동(예: 가드레일 LLM)하며, **한국어 가드모델은 차별화 여지가 크다**. 규모가 크므로 P0/P1 이후 별도 설계로 분해한다.

### P3 — 킬스위치·자산 무결성 (S~M)

25(긴급 정지)·⑭·⑰(자산 목록·모델 파일 무결성)을 명시적 기능으로 정리한다. 현재 우회 차단·감사 무결성으로 부분 충족하는 것을 운영 안전장치로 격상한다.

## 4. 범위 밖 (Won't — 초점 보호)

다음은 NuFi가 **다루지 않는다**(파트너·이연):

- 판단형 적대적 robustness 검증(⑨ 판단형·⑩·⑬)
- 데이터·모델 오염 공격(⑮·⑯·⑱·⑲)
- 공급망 SBOM·외부 도구 검증(⑳·21·22·37·38)
- AI 보안 교육(28·망 1.4)
- 제3자 독립 검증 프로세스(30)
- AI 관리자 지정·소비자 고지 등 조직 절차(망 1.1·1.3·1.5·1.6)

이를 떠안으면 제품 초점이 흐려진다. 매핑 리포트(P0)에서는 이들을 "범위 밖"으로 **투명하게 표기**하여, 고객이 잔여 항목을 별도 절차로 충족하도록 안내한다.

## 5. 시퀀싱

```
P0 (매핑 리포트, S) ──▶ P1 (rate limit, M) ──▶ P2 (회피 공격 탐지, L)
                                          └──▶ P3 (킬스위치·자산 무결성, S~M)
```

P0를 먼저 출하해 규제 증빙 가치를 확보하고, 그 매핑이 드러내는 부분 충족 항목(P1~P3)을 동일 패턴으로 순차 분해한다.

## 6. 거버넌스 · 리스크

- **설계·구현 분리:** 설계·명세는 제품 측이, 구현 트랙은 **사람 개발자**가 오너로 맡는다.
- **목표(OKR) 연결 공백:** 본 로드맵 항목은 아직 회사 목표·핵심결과에 연결되어 있지 않다. P0가 규제·도입 결정에 직결되므로 목표 연결 1순위 후보다 — 리더십 결정이 필요하다.
- **확정 절차:** 우선순위·자원 배분은 리더십/보드 정합 후 확정한다. 본 문서는 그 정합을 위한 제안이다.
