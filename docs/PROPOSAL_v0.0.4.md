## 0. 출발점 — v0.0.3(Operate) 가 끝낸 것 / 남긴 것 / 이미 쌓인 것

**끝남(Operate M1 — 관측·보증, v0.0.3 RELEASED 2026-06-28, tag `v0.0.3`):**
- O1 감사 가시성 대시보드(read-only, 4패널: 결정/해시체인/우회/카테고리) — CMP-134

- O1·O2 1-명령 데모 + 재현 매뉴얼 — CMP-138

**🟡 이미 쌓였으나 미릴리스([Unreleased] — local main 4 ahead of origin, v0.0.3 태그 이후):**
- **CMP-141** 설치형 `console_scripts` 패키징(`nufi-egress`/`nufi` = 실제 PATH 진입점) + `audit` 서브커맨드 통합(§4 집계 = `audit query`). *— 기능 트랙(feat). 이미 머지됨.*
- **CMP-140** INTEGRATION_GUIDE 멀티-프로바이더(Claude/Anthropic·Google·Azure) 명시 + 운영 명령을 설치형 `nufi-egress` 리드로 재작성(문서 3커밋).

→ **함의:** v0.0.4 는 *빈손에서 시작하지 않는다.* CMP-141(설치형 패키징)은 그 자체로 사용자 대면 가치이며 **현재 어떤 태그에도 실리지 않은 상태.** 방치 시 "pip 설치형 진입점" 이 무기한 미릴리스로 남는다.

**남김(Operate 미완 + 누적 부채 — 본 제안의 표적):**
- **O3 정책 운영 규모화 = 미릴리스** — v0.0.3 에서 *M2 백로그로 명시 이연.* 핫리로드는 여전히 단건/단일프로파일(CMP-124). 멀티프로파일·라우트/테넌트별 바인딩·버전/롤백 미지원 → **다중 워크로드 운영 불가.**
- **정확도 부채(M6) = 두 번 이연** — INT8 KR_PERSON Wilson CI 하한 **0.832** 가 목표 0.85 를 ~1.8%p 하회(소표본 양자화 노이즈). v0.0.1 'Known limitations' → M6 이연, v0.0.3 도 미착수. *유일하게 남은 측정 게이트 잔여.*
- **프로덕션 온프렘 p95 미실측** — 38ms 는 개발 환경 INT8 측정치. 온프렘 실측은 M6 후속으로만 표기됨.
- **O4 IDM 문서지문(DLP 깊이)** — 디자인파트너 수요 신호 의존(v0.0.2·v0.0.3 모두 조건부 보류).

→ **결론:** v0.0.3 이 *"보안팀이 온프렘에서 관측·보증"* 을 풀었다면, v0.0.4 의 표적은 **"운영자가 정책을 안전하게 규모 있게 굴린다(O3 완성)"** + **"두 번 미룬 정확도 부채를 종결"** + **이미 머지된 설치형 패키징을 정식 릴리스.** 즉 v0.0.4 = *Operate 호라이즌의 완결 + 누적 위생 청산.* 그 다음이 v0.1.0 Scale·Differentiate.

## 1. v0.0.4 후보 트랙 5종 (보드가 조합 선택)

각 트랙 = 무엇 / 왜 / 대표 산출물 / Effort / 오너. 구현 오너=Engineer(CMP-96), **릴리스 권한 행위(태그/VERSION 컷)는 사람**(CMP-60/137).

### A0 — Release Capture (이미 머지된 것을 정식 릴리스) ★선행/위생
- **무엇:** [Unreleased] 의 CMP-141(설치형 패키징·`audit` 서브커맨드) + CMP-140(멀티프로바이더 문서)을 v0.0.4 CHANGELOG 섹션으로 포착 → `VERSION`→0.0.4 · CHANGELOG [0.0.4] · `git tag v0.0.4` + 푸시. (단독으로도 patch 릴리스 성립.)
- **왜:** 설치형 진입점은 사용자 대면 가치인데 **현재 어떤 태그에도 없음.** 베이스라인을 고정해야 위에 dev 를 쌓는다.
- **대표 산출물:** CHANGELOG [0.0.4] 초안(CPO 설계 가능) + RELEASE_CHECKLIST 통과 + 태그 컷(사람).
- **Effort:** S · **오너:** **사람 릴리스 오너**(CMP-60/137 — AI 비배정)

### A1 — Policy Ops at Scale (정책 운영 규모화 = O3 이월) ★dev 척추
- **무엇:** CMP-124 단건 핫리로드를 규모화 — ① **멀티프로파일** 동시 운영 + **라우트/테넌트별 프로파일 바인딩** ② 정책 **버전·드리프트·롤백**(무재기동) ③ 변경 감사(누가·언제·무엇을).
- **왜:** 단일 프로파일·단건 리로드로는 다중 워크로드 운영 불가. v0.0.3 에서 명시 이연된 *Operate 의 마지막 기둥.* 운영자가 정책을 안전하게 굴리는 1급 경험.
- **대표 산출물:** 프로파일 레지스트리 + 바인딩 규칙(`routing.yaml` 확장), 버전/롤백 명령(`nufi-egress policy …`), 변경 감사 로그.
- **Effort:** M · **오너:** Engineer (CMP-96)

### A2 — Accuracy Debt Closure (정확도 부채 종결 = M6)
- **무엇:** ① NER base 모델 격상으로 **INT8 KR_PERSON Wilson CI 하한 ≥ 0.85** 달성(현 0.832) ② **프로덕션 온프렘 p95 재실측**(현 38ms 는 개발 INT8). 격상 후 INT8 requant 정합성 검증 동반.
- **왜:** 두 번 이연된 *유일한 잔여 측정 게이트.* 종결 시 0.0.1 'Known limitations' 의 핵심 1건 제거 → 측정 신뢰 완성. 단 모델 격상은 평가·재양자화 비용 있음(Effort 변동).
- **대표 산출물:** 격상 모델 벤치 리포트(M5 하니스 재사용) + INT8 정합성 테스트 + 온프렘 p95 표.
- **Effort:** M–L · **오너:** Engineer (CMP-96)

### A3 — Detection Depth / IDM DLP (탐지 깊이 · 수요 조건부)
- **무엇:** 기밀 3계층 **IDM 문서지문**(인덱싱된 문서 일치 — EDM 비구조화 확장). 임베딩 분류기는 그 다음.
- **왜:** 차별화된 DLP. **단 디자인파트너 수요 신호 의존** — 미확인 선투자 비권장(v0.0.2·v0.0.3 모두 보류).
- **대표 산출물:** (수요 확인 시 별도 설계) IDM 인덱서 + 매칭기.
- **Effort:** L–XL · **오너:** Engineer (수요 확인 후)

### A4 — Release Automation (릴리스 자동화 · 위생 후속)
- **무엇:** 현재 수작업인 VERSION/CHANGELOG/태그/체크리스트 컷을 스크립트화(드라이런 + 검증 게이트).
- **왜:** v0.0.2~0.0.4 릴리스가 매번 수작업. 자동화로 위생 부채 재발 방지. **단 릴리스-컷 도구 구현은 권한 행위 인접** → 설계는 CPO, 구현·실행은 사람 게이트 유지.
- **대표 산출물:** `scripts/release.sh`(드라이런) + 체크리스트 자동검증.
- **Effort:** S–M · **오너:** 사람(릴리스 권한 인접) — 우선순위 낮음, v0.1.0 으로 이연 권장.
